# Skill: 初始化控制文档

## 目标

在项目刚开始或docs下控制文档缺失时，优雅创建并初始化项目控制文档。

## 适用场景

当用户请求开发、修改、生成计划、更新项目文件时，如果控制文档不存在、为空，必须先执行本 skill。

## 需要创建的文件

- `docs/ARCHITECTURE.md`
- `docs/SCHEDULE.md`
- `docs/CHAT.md`
- `docs/RULE.md`
- `docs/CLOUD.md`

## 初始化规则

1. 如果 `docs/` 目录不存在，创建 `docs/`
2. 如果控制文档不存在，使用`skills/init-control-docs/templates/`下模板创建对应文件
4. 如果模板不存在，暂停流程，并指出
5. 初始化内容只能基于当前项目真实状态，如果是空目录，则在需要补充的地方，写入带确定
6. 无法确认的信息写为“待确认”
7. 初始化完成后写入标记："已由 AI 根据当前项目状态初始化"


## 禁止行为

- 禁止猜测用户意图
- 禁止猜测不存在的接口
- 禁止猜测数据库结构
- 禁止把未确认的用户想法写入正式控制文档
